프로젝트명 : 여신 티켓 (Spring  - 데이터베이스 기반)

투입인원 : 1명

프로젝트 소개 : 피부과 탐색 및 예약이 가능하며 사람들의 후기를 확인할 수 있는 웹 사이트 


사전 작업 :
1. 데이터베이스 기획 및 구현
2. 필요한 자료 수집

기능 구현
1. 페이지 기능
 페이지버튼이 필요한 화면에다가 삽입
 출력할 게시물의 개수에 비례해서 필요한 페이지 버튼이 생성 
     Ex :  1  or  1 2 3 4 5  
 삼각형을 클릭 시 이전 혹은 다음 페이지로 이동
 숫자를 클릭 시 해당 페이지로 이동

2. 피부과(병원) 목록 페이지
 보유한 병원 목록 출력
 버튼을 통해 여신TV/시술후기/피부과(병원)로 각 페이지 이동 가능
 페이지 버튼을 이용하여 페이지 이동 가능 : 1번 기능

3. 시술 후기 목록 페이지
 보유한 시술 후기 목록 출력
 버튼을 통해 여신TV/시술후기/피부과로 각 페이지 이동 가능
 원하는 시술 후기 클릭 시 해당 시술 후기의 상세페이지로 이동: 4번 기능
 페이지 버튼을 이용하여 페이지 이동 가능 : 1번 기능
 로그인이 되어있는 경우만 글쓰기 버튼 출력하도록 if문 활용
 글쓰기 버튼 클릭 시 시술 후기 작성 페이지로 이동  : 4번 기능
 로그인 시에만 접속되도록 interceptor 및 configuration 설정 :  19번 기능

4. 시술 후기 작성 페이지
 제목, 내용, 파일 업로드 가능
 파일 업로드 버튼 클릭 시 파일 업로드 가능 : 5번 기능
 선택된 파일들의 파일명 및 개수 출력
 작성자는 로그인한 고객의 정보(세션에 저장)로 hidden 처리
 Date는 Timestamp를 활용하여 현재 날짜로 처리
 글쓰기 완료 버튼을 통해 리뷰 상세 페이지로 이동 : 6번 기능
 Back Button을 통해 리뷰 목록 페이지로 이동 : 3번 기능

5. 파일 업로드 기능
 파일 업로드 시 webapp의 upload폴더에 저장
 폴더가 없을 경우 폴더 생성 처리
 해당 리뷰 작성 버튼 클릭 시 해당 리뷰의 기본키를 외래키로 데이터 베이스에 삽입
 삭제  /  삽입 / 수정은 Transaction을 활용 : 13번 기능
 파일의 이름은 중복 가능성을 염두 하여 밀로 초를 활용하여 새로운 이름으로 저장
 파일의 경로를 realPath를 활용하여 저장

6. 시술 후기 상세 페이지
 시술 후기의 기본키로 Mapper을 통해 데이터 베이스로부터 해당 정보 출력
 Update, Delete, Back 버튼을 통해서 해당 기능 구현
 Update 클릭 시 : 시술 후기 수정 페이지로 이동 : 7번 기능
 Delete 클릭 시 : 메시지 출력 후 확인 시 데이터 베이스와 목록에서 삭제 : 12번 기능
 Back 클릭 시 : 시술 후기 목록 페이지로 이동 :  3번 기능
 조회수 기능 : 17번 기능

7. 시술 후기 수정 페이지
 기본키를 활용하여 시술 후기 상세 페이지에 업로드할 정보를 수정하여 재 업로드 및 데이터 베이스에 업데이트
 삭제  /  삽입 / 수정은 Transaction을 활용 : 14번 기능

8. 회원가입 페이지
 아이디 중복 체크 ajax를 활용하여 데이터 베이스로 데이터 전송 및 비교 후 메시지 출력
 모든 input에 데이터가 작성되었는지 확인 후 로그인 페이지로 이동 : 8번 기능
 비밀번호는 암호화 처리 후 데이터 베이스에 저장
 작성되어진 데이터들의 value값을 Mapper을 통해 데이터 베이스에 삽입
 삭제  /  삽입 / 수정은 Transaction을 활용 : 13번 기능

9. 로그인 페이지
 로그인 화면에 아이디 및 비밀 번호 입력
 아이디를 Mapper을 통해 비교 후 해당 아이디에 해당하는 암호화된 비밀번호를 비교
 일치할 경우 로그인 정보를 세션에 저장 후 메인 화면으로 이동 : 18번기능
 불일치할 경우 메시지 출력 후 로그인 화면 지속

10. 회원 정보 출력 페이지
 로그인한 고객의 아이디(세션에 저장)를 Mapper을 통해 데이터 베이스에서 정보를 가져와서 출력
 SELECT 쿼리 문 사용

11. 여신TV 페이지
  피부 관리 동영상들 video로 출력

12. 삭제 / 취소 기능
 해당 정보의 기본키를 Mapper을 통해 데이터 베이스에서 비교하여 해당 정보를 삭제
 DELETE 쿼리 문 사용

13. 삽입 / 저장 기능
 저장하고 싶은 정보들을value값으로 넘겨서Mapper을 통해 데이터베이스에 삽입
 조건 추가 가능
 INSERT쿼리 문 사용

14. 업데이트 기능
 수정하고 싶은 정보들을value값으로 넘겨서Mapper을 통해 데이터 베이스에 수정
 조건 추가 가능
 UPDATE쿼리 문 사용

15. 댓글 작성 기능
 댓글 입력 칸을 input으로 처리
 댓글 저장 버튼 클릭 시 해당 리뷰의 기본키를 외래키로 데이터 베이스에 삽입
 삭제  /  삽입 / 수정은 Transaction을 활용 :  13번 기능

16. 검색 기능
 작성되어진 키워드를 Mapper을 통해 데이터 베이스 데이터와 Like와 와일드카드를 활용하여 비교함으로써 해당되는 게시물만 출력
 SELECT 쿼리 문 사용

17. 조회수 기능
 로그인하지 않은 경우나 로그인한 회원의 정보와 게시물 작성자의 정보가 같지 않을 경우 조회수 증가
 비교하기 위해 if문 활용 

18. 메인 화면
 bxSlider을 활용하여 사진 출력
 해당 부분들에 맞게 클릭 시 해당 페이지 이동 설정

19. Interceptor & Configuration
 로그인을 하지 않은 경우 시술 후기 목록 등 몇몇 페이지로 이동 시 에러 발생
 이를 방지하기 위해 로그인한 경우에만 해당 페이지 접속이 되도록 설정
 로그인하지 않은 경우 해당 페이지 이동 버튼을 클릭 시 로그인 페이지로 redirect 설정

20. HTML 및 CSS
 오픈 소스 사용하지 않고 직접 제작

개발도구 :

∙ 언어 - JAVA  / HTML / CSS / JQuery / JavaScript
∙ IDE 도구 - IntelliJ
∙ DB - MySQL
